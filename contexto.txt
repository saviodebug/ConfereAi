Contexto do Projeto VerificaVoto

  Estou desenvolvendo duas extensões Chrome Manifest V3 chamadas VerificaVoto.

  ## 1. VerificaVoto grátis/local

  Pasta:
  C:\Users\Administrator\verificavoto-gratis

  Características:
  - Extensão Chrome Manifest V3.
  - Sem backend.
  - Sem banco de dados.
  - Sem API externa paga.
  - Usa HTML, CSS e JavaScript puro.
  - Analisa notícias eleitorais por regras locais em rules.js.
  - Captura título, URL e texto visível da página.
  - Também aceita:
    - print/imagem com OCR local usando Tesseract.js;
    - colar print com Ctrl+V;
    - colar texto manualmente.
  - Não afirma que uma notícia é falsa.
  - Apenas aponta sinais de baixo, médio ou alto risco.

  Problemas já corrigidos:
  - Autor do G1 não era detectado corretamente.
  - G1 era marcado como URL suspeita indevidamente.
  - Datas em notícias reais precisaram ser detectadas melhor.
  - OCR local precisou usar arquivos locais, sem CDN.
  - Popup passou a aceitar Ctrl+V para print.
  - Popup passou a aceitar texto colado manualmente.
  - Foram adicionadas logos no projeto.

  Arquitetura:
  Extensão Chrome → Backend Node.js local → SQLite → Gemini API opcional → Resposta para extensão

  Tecnologias:
  - Chrome Extension Manifest V3
  - HTML, CSS e JavaScript puro
  - Node.js
  - Express
  - CORS
  - dotenv
  - SQLite
  - @google/genai
  - Tesseract.js local para OCR

  Funcionalidades da extensão AI:
  - Analisar página atual.
  - Analisar print/imagem.
  - Colar print com Ctrl+V.
  - Analisar texto colado manualmente.
  - Enviar dados para o backend local:
    POST http://localhost:3000/analisar
  - Mostrar:
    - classificação;
    - pontuação;
    - sinais encontrados;
    - critérios aplicados;
    - palavras-chave eleitorais;
    - análise da IA, se usada;
    - fontes sugeridas;
    - recomendação;
    - histórico recente;
    - estatísticas;
    - botão copiar relatório;
    - botão gerar relatório HTML;
    - dashboard com histórico.

  Backend:
  - Porta padrão: 3000.
  - Rotas:
    - GET /health
    - POST /analisar
    - GET /historico
    - GET /fontes
    - GET /estatisticas

  Banco SQLite:
  - Tabela analises.
  - Tabela fontes_confiaveis.
  - Tabela palavras_chave.

  Fontes confiáveis cadastradas:
  - TSE
  - Fato ou Boato
  - TREs
  - Agência Lupa
  - Aos Fatos
  - Estadão Verifica
  - Agência Brasil
  - STF
  - Senado
  - Câmara dos Deputados

  Regras locais:
  - ausência de autor: +2
  - ausência de data: +1
  - linguagem alarmista: +2
  - pedido de compartilhamento rápido: +2
  - alegação grave sobre fraude eleitoral sem fonte oficial: +3
  - ausência de fonte oficial: +2
  - muitos termos em caixa alta: +1
  - URL suspeita: +1
  - domínio jornalístico/institucional conhecido: -1
  - fonte oficial citada: -2
  - autor e data encontrados: -1

  Classificação:
  - 0 a 2 pontos: Baixo risco
  - 3 a 6 pontos: Médio risco
  - 7 ou mais: Alto risco

  Gemini:
  - A chave fica somente em:
    C:\Users\Administrator\verificavoto-ai-gemini\backend\.env
  - Nunca colocar chave no frontend.
  - Nunca colocar chave no Git.
  - .env.example existe como modelo.
  - Se Gemini falhar, o backend continua com regras locais.
  - A classificação principal sempre vem das regras locais.
  - Gemini só complementa a explicação.

  Variáveis do .env:
  PORT=3000
  USE_GEMINI=true
  USE_GEMINI_GROUNDING=false
  GEMINI_API_KEY=minha-chave-aqui
  GEMINI_MODEL=gemini-flash-latest
  GEMINI_FALLBACK_MODEL=gemini-3.5-flash

  Problemas já encontrados:
  - Gemini retornou erro 503:
    “This model is currently overloaded”.
  - Foi implementado fallback de modelo.
  - gemini-1.5-flash deu 404 em v1beta.
  - Foi usado fallback com modelos mais atuais.
  - Gemini às vezes pode errar análise temporal, dizendo que data de 2026 é futura.
  - Foi corrigido o prompt para enviar a data atual como referência.
  - Foi instruído que o Gemini não deve tratar domínio jornalístico conhecido como simulação sem indício claro.
  - Foi adicionada limpeza da resposta da IA para remover Markdown como **asteriscos**.

  Versão atual da extensão AI:

  Como rodar:
  1. Abrir terminal:
     cd C:\Users\Administrator\verificavoto-ai-gemini\backend

  2. Instalar dependências, se necessário:
     npm install

  3. Rodar backend:
     npm run dev

  4. Abrir Chrome:
     chrome://extensions

  5. Ativar modo desenvolvedor.

  6. Carregar sem compactação:
     C:\Users\Administrator\verificavoto-ai-gemini\extensao

  7. Ao alterar código:
     - reiniciar backend se mexeu no backend;
     - recarregar extensão se mexeu na extensão.

  Observações importantes:
  - A ferramenta não determina se uma notícia é verdadeira ou falsa.
  - Ela apenas aponta sinais de risco e recomenda checagem.
  - IA pode errar, principalmente em contexto temporal, datas e interpretação.
  - OCR pode errar se a imagem estiver borrada, cortada ou com texto pequeno.
  - Para apresentação acadêmica/PTT, destacar:
    - versão gratuita por regras locais;
    - versão avançada com backend, SQLite e Gemini opcional;
    - privacidade da chave no backend;
    - fallback sem IA;
    - transparência dos critérios;
    - histórico e relatórios.
